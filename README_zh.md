# Super-Prompt：引导式提示词精铸

[![License: MIT](https://img.shields.io/badge/License-MIT-blue.svg)](LICENSE)
[![Standard: Agent Skills](https://img.shields.io/badge/Standard-Agent%20Skills-6e3ef7.svg)](https://agentskills.io)

> 🎯 **让 AI 收到的每个指令都精准无歧义——而不是"差不多就行"。** — 反歧义拦截 × 联网预搜索 × 分层引导 × ELO 排名，从一句话到生产级提示词。

**提示词工程有一个根本性缺陷：模型会学会绕过提示词。** 你说"强制验证"，它输出四个字："强制验证通过"。你说"详细说明"，它写 3 行废话也算详细。你说"用 JSON 格式"，它自由发挥字段名。

Super-Prompt 不是模板，不是填空表，是**一整套提示词纪律体系**。通过七道闸门引导式交互 + 联网搜索预加载 + 反歧义校验 + ELO 排名，帮用户从一句话或一份产品文档，精铸出 **AI 无法误解的生产级提示词**。

## 快速开始

```bash
npx skills add wlj103/super-prompt
```

触发方式：说"帮我写一个提示词" / "帮我优化这段 prompt" / "精铸提示词"

## 工作原理

### 七道闸门

| 闸门 | 防止什么 |
|------|---------|
| **G0: 意图澄清** | 用户目标模糊，AI 瞎猜 |
| **G1: 领域搜索** | 缺少行业知识、最佳实践、已知坑点 |
| **G2: 反歧义锁定** | 18 条清单逐条扫描，禁止空口通过 |
| **G3: 结构化草稿** | 强制章节、精确约束、零自由发挥 |
| **G4: 对抗测试** | 模拟 AI 可能误解的 3 种路径，加固弱点 |
| **G5: ELO 排名** | 多候选时 ELO 锦标赛排名（K=32，起跑 1200） |
| **G6: 交付** | 最终提示词 + 速查卡 + 维护义务 |

### 输出复杂度：轻 / 中 / 重

Super-Prompt 自动判断项目复杂度，调整输出深度：

- **轻级** — 一句话优化，闸门一规则引擎，不调 LLM
- **中级** — 有明确场景但 prompt 不完整，G1→G3→G5，标准 ELO
- **重级** — Agent 系统提示词、生产级 prompt，全七道 + 18 条反歧义 + ELO×2

### 输入模式

- **标准** — 互动问答，每轮 ≤4 问
- **审查** — 审计已有提示词，逐条过反歧义清单
- **重型** — 多 Agent 系统、SOP、复杂工作流

输入模式与输出复杂度正交——可以标准输入 + 重级输出。

## 设计原则

1. **预搜索，不猜测** — 每问必搜，搜索结果转化为选项注入
2. **分层引导，不审问** — 每轮 ≤4 问，从最简单必选问题开始
3. **反歧义，不假设** — 18 条清单，系统锁定每条歧义路径
4. **证据驱动，不观点驱动** — 设计决策有搜索证据或行业准则支撑
5. **ELO 验证，不人工判断** — 多候选 ELO 锦标赛排名
6. **交付带保险** — 速查卡 + 维护义务 + 降级与异常处理
7. **按需输出，不过度** — 不偷工减料，也不过度输出，安全边际只增不减

## super-spec × super-prompt

Super-Prompt 是 [super-spec](https://github.com/wlj103/super-spec) 的姐妹项目：

| | super-spec | super-prompt |
|---|---|---|
| **问题** | AI 跳步骤，零证据声称成功 | AI 表达模糊，缺乏结构，留下歧义 |
| **方案** | 工程纪律 — 7 闸门、5 文档、反偷懒 | 提示词纪律 — 7 闸门、18 条反歧义、ELO 排名 |
| **领域** | Agent 如何**执行** | Agent 如何**表达** |

两者合在一起，形成完整的纪律层：一个确保 Agent **做对的事**，一个确保 Agent **用对的方式表达**。
